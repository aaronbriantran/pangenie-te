submitting multiple jobs at once as an array (taken from https://hpcdocs.hpc.arizona.edu/running_jobs/batch_jobs/array_jobs/#example-jobs)

first, in the directives (top of bash script) we can add:

#SBATCH --array=1-10

like any subset, you can make this jump around (e.g. 1,3,5..., can even use modulo logic)

this sets the ${SLURM_ARRAY_TASK_ID} variable in the bash script. to get it to retrieve a name from a list, we can use ls and sed.

ls -l <file_directory> > inputs.txt
sed "${SLURM_ARRAY_TASK_ID}q;d" inputs.txt [this returns the line of text corresponding to the ID in the inputs.txt file]

naturally we can subshell this in our script:

INPUTNAME=$( sed... ) #use the task id!
OUTPUTNAME=$( some awk or sed operation on the input to get the output name. use basename, etc.)
pangenie -i $INPUTNAME -o $OUTPUTNAME

this submits a bunch of different jobs under the array type, which can be individually allocated! :)
