After some thinking, I revised the Snakefile for the QC step so that it aggregates all of the fastas in the fastqc rule. That way, we don't have to worry about aggregating the outputs by forcing them to write to the same directory.

Instead, using the expand() function creates a bunch of filenames with spacing between them, which can be passed as multiple spaced out arguments to fastqc. I'm pretty sure expand() returns a list, but that Snakemake handles converting lists into strings when passed.
CITE: https://farm.cse.ucdavis.edu/~ctbrown/2023-snakemake-book-draft/beginner+/expand.html
The documentation for fastqc can't be cited, but I confirm that fastqc --help has the behavior I described.

So the new flow:
* I pass a Python list of sample names
* I expand() it in the FastQC rule
* The expand() list is converted into space-delimited strings
* This is passed as an argument to the script
* The script takes the argument, which actually passes a space-delimited set of strings
* FastQC happens all at once with multiple arguments
