bold steps are things you need to ask Juan about

reads -> count kmers -> kmers -> Pangenie genotype imputation -> VCF -> SHAPEIT5 + pedigree info -> phased VCF
-> prune VCF of unphased sites -> pruned VCF -> bcftools consensus -> haplotigs -> construct shared graph
-> align to individual haplotigs -> hap-specific alignments -> align to shared graph -> shared alignment -> categorize reads in shared alignment
-> subtract reads from hap-specific alignments -> call variants from hap-specific alignments
-> bcftools consensus for hap-specific alignments -> hap-specific personalized alignments

the issue is that lineage-based phasing works off of variants. it can only use rare variants.
- VCF -> stat phasing -> phased VCF
- reads + haplotigs -> read phasing -> read-based phasing




1. A set of variants describes a contig.
2. A contig determines alignments.
3. Alignments inform a set of variants.

Phasing is simply an additional standard for the precision of a contig to the molecule we are measuring.

How do we start the cycle?
	- Instead of aligning, we impute directly from a reference (3). This generates a fuzzy callset of variants.
	- We then phase those variants using statistical and lineage information. This generates two sets of phased variants.
	- Each set of variants then describes a haplotig (1), which is precise to the level of haplotype.
	- We then align to individual and pooled haplotigs (2). We recognize that the difference in contig contexts creates differences in alignments, which we use to phase reads.
	- We now have two sets of reads, each determined by phased contigs.
	- We use each set of reads to inform phased variants (3).

Increasing levels of alignment precision
Consensus reference -> genotype imputed reference -> relatives reference -> personal reference
