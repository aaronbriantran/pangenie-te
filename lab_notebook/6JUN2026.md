Added a README to the fastqc step. In general, READMEs should contain:
* author
* most recent update day
* purpose of the step
* modules used, along with versioning (use module spider to check for ml)
   * for downloaded executables, links to download a given binary
   * for python based libraries, direct to the config file for environment.yml to construct conda envs per step
* inputs/outputs

In general, this helped me solidify a few things:
* Snakemake should be used per step. It can handle changing SLURM parameters per rule that it runs, so it will be a runall
* environment.yml files should be generated per step, using conda to download and produce these files
