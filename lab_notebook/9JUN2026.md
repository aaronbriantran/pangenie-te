I wrote a MultiQC script. This script spins up its own conda environment based off the environment yaml file for that step if it doesn't exist. This basic control of conda environments works by checking <code>conda env --list</code> using grep for the name of the expected environment.

The script then goes into the results directory of FastQC and summarizes all of the files there.

I wrote the multiqc.yml file that creates the multiqc environment. I've confirmed with a run on the old servers that this does work. However, it turns out that conda's docs for the command for how to create an environment are wrong, and you need to use <code>conda env create --file multiqc.yml</code>. I've modified this logic in multiqc.sh.
Cite: https://stackoverflow.com/questions/56871882/condavalueerror-the-target-prefix-is-the-base-prefix-aborting

I haven't worked on writing down the multiqc version because I'm not actually sure which version of MultiQC that conda will download. Ideally, of course, it will be the most recent one. The link below includes a general syntax for an environment.yml file and includes a description for how to restrict the versioning, if you want to be even more reproducible.
Cite: https://rse.shef.ac.uk/conda-environments-for-effective-and-reproducible-research/04-sharing-environments/index.html
